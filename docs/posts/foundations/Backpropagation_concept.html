Backpropagation (역전파) — 딥러닝 학습의 핵심

모든 딥러닝 모델이 학습되는 원리. CNN, Transformer, Diffusion 모두 이 알고리즘 위에서 작동한다.

한 줄 요약

모델의 오차가 각 파라미터에 얼마나 영향을 주는지 chain rule로 계산하는 알고리즘이다.

전체 흐름

forward pass (예측)
    → loss 계산
        → backward pass (gradient 역전파)
            → optimizer로 weight 업데이트
                → 반복

이 문서를 읽고 설명할 수 있어야 하는 핵심 3가지

  • chain rule이 역전파에서 어떻게 쓰이는지
  • vanishing gradient가 왜 생기고, ReLU와 residual connection이 어떻게 해결하는지
  • PyTorch의 loss.backward()가 내부적으로 무엇을 하는지

처음 보는 사람용 핵심 용어

  • Gradient: 파라미터를 조금 바꿨을 때 loss가 얼마나 변하는지. 방향과 크기를 모두 담고 있다.
  • Chain Rule (체인룰): 합성함수의 미분 법칙. f(g(x))의 미분 = f'(g(x)) · g'(x). 역전파의 수학적 핵심.
  • Activation: 각 layer의 출력값. forward pass 중 저장해두고 backward에서 재사용한다.
  • Computation Graph: 연산들의 연결 구조. PyTorch가 내부적으로 만들어서 자동으로 역전파에 쓴다.
  • Vanishing Gradient: layer가 깊을수록 앞쪽 gradient가 점점 0에 가까워지는 현상.

1단. 왜 역전파가 필요한가

신경망 학습의 목표는 loss를 최소로 만드는 파라미터 w를 찾는 것이다.

y = f(x, w)          # 신경망의 예측
L = loss(y, y_star)  # 예측과 정답의 차이
# 목표: min L (w를 조절해서)

이를 위해 각 파라미터가 loss에 얼마나 영향을 주는지 알아야 한다 → ∂L/∂w (gradient).

신경망이 1~2층이면 직접 미분해도 되지만, 수십~수백 층이면 직접 계산이 불가능하다. → chain rule을 이용한 역전파 필요.

2단. Chain Rule — 역전파의 수학적 핵심

합성함수의 미분은 각 부분의 미분을 곱한다.

L = f3(f2(f1(x)))

# chain rule:
dL/dx = (dL/df3) * (df3/df2) * (df2/df1) * (df1/dx)
#       ← 뒤에서부터 곱해나간다 →

핵심은 한 번 계산한 gradient를 재사용한다는 것이다. dL/df3를 계산해두면, dL/df2를 계산할 때 그 값을 그대로 쓴다. 이 덕분에 전체 계산 비용이 forward pass와 비슷한 수준이다.

3단. 실제 학습 흐름

# 1. Forward pass: 예측값과 중간 activation 계산 (저장)
z1 = W1 @ x          # layer 1
z2 = W2 @ z1         # layer 2
y  = W3 @ z2         # layer 3 (예측)

# 2. Loss 계산
L = ((y - y_star)**2).mean()

# 3. Backward pass: loss에서 시작해 gradient 역전파
dL_dW3 = dL/dy   * dy/dW3            # W3 gradient
dL_dW2 = dL/dz2  * dz2/dW2          # W2 gradient (dL/dz2는 위에서 재사용)
dL_dW1 = dL/dz1  * dz1/dW1          # W1 gradient

# 4. Optimizer로 업데이트
W1 -= lr * dL_dW1
W2 -= lr * dL_dW2
W3 -= lr * dL_dW3

4단. 간단한 수식 예 (단층)

모델: y = wx, Loss: L = (wx - y*)²

# Forward
y = w * x

# Gradient 계산
dL/dw = 2 * (w*x - y_star) * x

# Update
w = w - lr * dL/dw

직관: (wx - y*)는 오차의 크기, x는 w가 y에 미치는 영향. 둘을 곱하면 w가 loss에 미치는 영향이 나온다.

핵심 수식 정리

① Chain Rule (기본)

dL/dx = (dL/df) * (df/dg) * (dg/dh) * (dh/dx)
# 합성함수: 각 단계의 미분을 곱한다

② 다층 linear network

dL/dW3 = dL/dz3 * z2.T
dL/dz2 = W3.T @ dL/dz3        # 재사용
dL/dW2 = dL/dz2 * z1.T
dL/dz1 = W2.T @ dL/dz2        # 재사용
dL/dW1 = dL/dz1 * x.T

한 번 계산한 dL/dz를 다음 layer에 전달한다. 이것이 재사용의 핵심.

③ Vanishing Gradient (sigmoid 예)

sigmoid'(x) = sigmoid(x) * (1 - sigmoid(x))  # 최대 0.25
# 10층이면: 0.25**10 ≈ 1e-6 → 앞쪽 layer gradient 거의 0

5단. 문제들과 해결 기술

Vanishing Gradient → ReLU로 해결

ReLU(x) = max(0, x)
ReLU'(x) = 1 if x > 0 else 0   # 양수 구간에서 gradient = 1 (사라지지 않음)

gradient가 1로 유지되어 layer를 거쳐도 사라지지 않는다. 이 하나의 변화가 딥러닝 혁명을 만들었다.

깊은 네트워크 학습 → Residual Connection

output = F(x) + x   # skip connection

# gradient:
dL/dx = dL/d_output * (F'(x) + 1)
#                              ↑ 항상 1이 더해짐 → gradient 최소 1 보장

F'(x)가 0이더라도 +1이 있어서 gradient가 완전히 사라지지 않는다. ResNet, Transformer 모두 이 구조를 쓴다.

Exploding Gradient → Gradient Clipping

if grad_norm > max_norm:
    grad = grad * (max_norm / grad_norm)   # norm을 max_norm으로 제한

LLM, RNN 학습에서 표준으로 사용.

6단. PyTorch에서의 자동 역전파

직접 구현할 필요 없다. PyTorch가 computation graph를 만들고 자동으로 처리한다.

x = torch.tensor([2.0], requires_grad=True)
w = torch.tensor([3.0], requires_grad=True)

y = w * x              # forward: computation graph 구성
L = (y - 5.0) ** 2    # loss

L.backward()           # 역전파 자동 실행
print(w.grad)          # dL/dw = 2(wx - y*) * x = 2*(6-5)*2 = 4.0

optimizer.zero_grad()  # 다음 iteration 전 gradient 초기화 (누적 방지)

동작 원리: 각 연산(`*`, `**` 등)이 자신의 gradient 함수를 graph에 등록한다. backward()가 graph를 역순으로 traversal하며 chain rule을 자동 적용한다.

예상 질문 5개와 답변

  • Q1. 역전파와 gradient descent의 차이는?
    역전파는 gradient를 계산하는 알고리즘, gradient descent(optimizer)는 그 gradient로 파라미터를 업데이트하는 방법이다. 역할이 다르다.
  • Q2. backward pass 비용이 forward보다 큰 이유는?
    activation을 저장하는 메모리 비용 + gradient 계산 비용이 추가된다. 일반적으로 backward ≈ 2 × forward 정도다.
  • Q3. ReLU의 Dead ReLU 문제는?
    음수 구간에서 gradient = 0이라서, 한 번 음수가 되면 다시는 학습되지 않는다. Leaky ReLU, GELU 등으로 부분 해결.
  • Q4. zero_grad()를 왜 매 iteration마다 해야 하나?
    PyTorch는 gradient를 누적한다. 초기화 없이 backward를 반복하면 이전 gradient가 더해져서 잘못된 방향으로 업데이트된다.
  • Q5. gradient checkpointing은 무엇을 트레이드오프하나?
    메모리 ↓ 대신 연산량 ↑. forward activation을 저장하지 않고 backward 시 필요한 부분을 재계산한다.
