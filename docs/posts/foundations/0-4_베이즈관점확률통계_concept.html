[기본기 0-4] 딥러닝을 위한 확률/통계 최소셋 (베이즈 관점)

난이도: 기초

태그: 확률통계,베이즈,우도,사후확률

딥러닝 학습의 본질은 "데이터가 주어졌을 때 어떤 파라미터가 가장 그럴듯한가"를 찾는 것입니다. 이를 확률로 쓰면 p(\theta|D)입니다.

왜 베이즈 관점이 중요한가

  • \theta: 모델 가설(파라미터)
  • D: 관측 데이터
  • p(\theta|D): 데이터를 본 뒤의 믿음(사후확률)

즉 데이터가 들어오면 모델에 대한 믿음이 업데이트됩니다.

1) 베이즈 정리

p(\theta|D) \propto p(D|\theta)p(\theta)

  • p(\theta|D): 사후확률 (posterior)
  • p(D|\theta): 우도 (likelihood)
  • p(\theta): 사전확률 (prior)

딥러닝에서는 계산 단순화를 위해 prior를 생략하고 MLE 관점(argmax_\theta p(D|\theta))으로 학습하는 경우가 많습니다.

2) 우도 (Likelihood)

p(D|\theta)는 "모델 파라미터가 주어졌을 때 데이터가 나올 가능성"입니다.

\theta^* = argmax\; p(D|\theta)

주의: 우도는 \theta에 대해 정규화된 확률밀도와는 해석이 다릅니다.

3) 로그우도 (Log Likelihood)

p(D|\theta)=\prod_i p(x_i|\theta)

\log p(D|\theta)=\sum_i \log p(x_i|\theta)

  • 곱셈을 덧셈으로 바꿔 계산이 쉬워짐
  • 수치 안정성 개선(아주 작은 값 언더플로우 완화)

4) 기댓값 (Expectation)

\mathbb{E}[X] = \sum_x x p(x)

딥러닝의 loss는 보통 데이터 분포에 대한 평균으로 해석됩니다.

\mathbb{E}[L(x)]

5) 분산 (Variance)

Var(X)=\mathbb{E}[(X-\mu)^2]

분산은 불확실성/퍼짐 정도를 나타내며, 정규화(BatchNorm)와 학습 안정성 해석에 중요합니다.

우도 vs 사후확률 (핵심 차이)

개념수식의미
우도p(D|\theta)파라미터가 주어졌을 때 데이터 확률
사후확률p(\theta|D)데이터가 주어졌을 때 파라미터 확률

코드-수식 연결

수식PyTorch 코드설명
p(D|\theta)model(x)모델의 데이터 설명력
\log p(x)torch.log(p)로그 확률
\sum \log p(x)log_prob.sum()로그우도 누적
\mathbb{E}[L]loss.mean()평균 loss

자주 하는 오해 5개

  • 우도와 확률을 완전히 동일하게 생각한다
  • prior를 항상 명시적으로 넣어야만 학습 가능하다고 생각한다
  • 로그우도를 단순 계산 트릭으로만 본다
  • 기댓값이 실제 샘플 평균과 항상 정확히 같다고 생각한다
  • 베이즈 관점은 실무 딥러닝과 무관하다고 생각한다
