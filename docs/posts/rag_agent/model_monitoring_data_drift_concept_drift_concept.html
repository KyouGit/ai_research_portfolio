주제: 6.5-5 Monitoring (Data Drift / Concept Drift / Prediction Monitoring)

분류: rag_agent

타입: concept

난이도: 중급

선수지식: 있음 — ML Pipeline, Model Serving

문제 설정

모델을 배포한 이후에도 성능이 항상 유지되는 것은 아닙니다.

시간이 지나면서 데이터 분포가 변할 수 있습니다.

예측 모델:

y = f(x; w)

기호 의미

  • x : 입력 데이터
  • w : 모델 파라미터
  • y : 예측 결과

문제는 실제 서비스 데이터 분포가 학습 데이터와 달라질 수 있다는 점입니다.

P_train(x) ≠ P_prod(x)

이를 감지하기 위해 필요한 것이 Model Monitoring입니다.

1. Monitoring 시스템 구조


User Request
     ↓
Model Prediction
     ↓
Logging
     ↓
Monitoring System
     ↓
Alert / Dashboard

모니터링 시스템은 모델 입력과 출력을 지속적으로 분석합니다.

2. Data Drift

Data drift는 입력 데이터 분포가 학습 데이터와 달라지는 현상입니다.

수식:

P_train(x) ≠ P_prod(x)

기호 의미

  • P_train(x) : 학습 데이터 분포
  • P_prod(x) : 서비스 데이터 분포

예

  • 사용자 행동 변화
  • 계절 변화
  • 새로운 데이터 유형

왜 중요한가

모델이 학습하지 않은 데이터가 증가하면 성능이 떨어집니다.

3. Concept Drift

Concept drift는 입력과 정답 사이 관계가 변하는 현상입니다.

수식:

P(y | x)_train ≠ P(y | x)_prod

기호 의미

  • P(y | x) : 조건부 확률

예

  • 스팸 필터 규칙 변화
  • 시장 패턴 변화
  • 사용자 행동 변화

왜 중요한가

모델 구조 자체가 더 이상 맞지 않을 수 있습니다.

4. Prediction Distribution Monitoring

모델 출력 분포도 모니터링해야 합니다.

출력 분포:

P(y_pred)

왜 필요한가

출력 분포 변화는 모델 문제의 신호일 수 있습니다.

예

  • 모든 예측이 동일 값
  • 확률 분포 붕괴
  • confidence 급변

5. Drift Detection Metric

분포 차이를 측정하기 위해 여러 metric을 사용합니다.

대표 metric:

KL(P || Q) = Σ P(x) log(P(x)/Q(x))

기호 의미

  • P : 기준 분포
  • Q : 현재 분포

왜 필요한가

분포 차이를 정량적으로 측정할 수 있습니다.

주의점

데이터 샘플 수가 충분해야 합니다.

6. Population Stability Index (PSI)

PSI는 데이터 분포 변화 감지에 자주 사용됩니다.

PSI = Σ (P_i - Q_i) log(P_i / Q_i)

기호 의미

  • P_i : 기준 분포 bin
  • Q_i : 현재 분포 bin

해석

PSI 값 의미
< 0.1 변화 없음
0.1 ~ 0.25 주의
> 0.25 drift 발생

자주 하는 오해 5개

  • 모델 배포 후 monitoring이 필요 없다고 생각한다
  • data drift는 드물게 발생한다고 생각한다
  • concept drift와 data drift는 동일하다고 생각한다
  • 모니터링은 로그만 저장하면 된다고 생각한다
  • drift가 발생해도 모델 성능에는 영향이 없다고 생각한다
