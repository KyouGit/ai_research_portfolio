주제: 6.5-4 Model Serving (Online Serving vs Batch Inference)

분류: rag_agent

타입: concept

난이도: 중급

선수지식: 있음 — ML Pipeline, Model Registry

문제 설정

모델이 학습된 이후 실제 시스템에서 사용하려면 Model Serving이 필요합니다.

즉 학습된 모델을 사용자 요청에 대해 실행하는 시스템입니다.

예측 함수:

y=f(x;w)

기호 의미

  • x : 입력 데이터
  • w : 모델 파라미터
  • y : 예측 결과

왜 필요한가

모델을 실제 서비스에서 사용할 수 있도록 합니다.

주의점

Serving 시스템은 latency, throughput, scalability를 고려해야 합니다.

1. Model Serving 구조


Client Request
     ↓
API Server
     ↓
Model Inference
     ↓
Response

서빙 방식은 크게 두 가지로 나뉩니다.

  • Online Serving
  • Batch Inference

2. Online Serving

Online Serving은 실시간 요청을 처리하는 방식입니다.

예:


User request → model prediction → response

특징

  • low latency 필요
  • request-response 구조
  • API 기반

통신 프로토콜

Protocol 특징
REST HTTP 기반 API
gRPC 고성능 RPC

예


POST /predict
{
  "input": "text"
}

왜 중요한가

사용자 인터랙션이 필요한 서비스에 필수입니다.

3. Latency 개념

Online serving에서 중요한 지표는 latency입니다.

Latency=Trequest+Tinference+Tresponse

기호 의미

  • T_request : 요청 전달 시간
  • T_inference : 모델 실행 시간
  • T_response : 응답 전달 시간

주의점

사용자 경험에 직접적인 영향을 줍니다.

4. Batch Inference

Batch inference는 여러 데이터를 한 번에 처리하는 방식입니다.


Dataset
   ↓
Batch processing
   ↓
Predictions

수식:

Y=f(X;w)

기호 의미

  • X : 데이터 집합
  • Y : 예측 결과 집합

왜 필요한가

대량 데이터 처리에 효율적입니다.

주의점

실시간 응답은 불가능합니다.

5. Online vs Batch

특징 Online Serving Batch Inference
Latency 낮음 필요 높아도 가능
처리 방식 실시간 주기적 처리
사용 사례 추천 시스템 데이터 분석

6. Triton Inference Server

NVIDIA Triton은 GPU 기반 모델 서빙 시스템입니다.

특징:

  • dynamic batching
  • multi-framework 지원
  • GPU 최적화

구조:


Client
 ↓
Triton Server
 ↓
Model Backend

7. TorchServe

TorchServe는 PyTorch 모델 서빙 시스템입니다.

특징:

  • REST API 지원
  • model version 관리
  • batch inference 지원

8. FastAPI Serving

FastAPI는 Python 기반 API 서버입니다.


from fastapi import FastAPI

app = FastAPI()

@app.post("/predict")
def predict(input):
    return model(input)

왜 사용되는가

간단한 ML 서비스에 많이 사용됩니다.

9. Serving 아키텍처


Load Balancer
      ↓
API Server
      ↓
Model Server
      ↓
GPU

핵심 목표

  • scalability
  • low latency
  • high throughput

코드-수식 연결

개념 코드 설명
API 요청 @app.post("/predict") REST endpoint
모델 실행 model(input) inference
batch 처리 model(batch) batch inference

자주 하는 오해 5개

  • 모델 학습과 serving은 동일한 환경이라고 생각한다
  • batch inference는 항상 느리다고 생각한다
  • REST API만 serving 방식이라고 생각한다
  • 모델 하나만 서버에 배포한다고 생각한다
  • serving 시스템은 확장이 필요 없다고 생각한다
