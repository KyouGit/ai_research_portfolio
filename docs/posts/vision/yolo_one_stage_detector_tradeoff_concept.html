주제: 7-2 YOLO (One-Stage Object Detector와 속도/정확도 Tradeoff)

분류: vision

타입: concept

난이도: 중급

선수지식: 있음 — CNN, Bounding Box, Classification

문제 설정

Object Detection의 목표는 이미지에서 객체 위치와 클래스를 동시에 찾는 것입니다.

예측 결과:

y = (class, bbox)

기호 의미

  • class : 객체 종류 (car, person 등)
  • bbox : bounding box 위치

기존 detection 모델은 두 단계 구조였습니다.


Region Proposal
      ↓
Object Classification

YOLO는 이를 한 번의 네트워크 실행으로 처리합니다.

이 구조를 One-Stage Detector라고 합니다.

1. YOLO 핵심 아이디어

YOLO는 이미지를 grid로 나누고 각 grid가 객체를 예측합니다.


Image
 ↓
Grid Split
 ↓
Bounding Box Prediction
 ↓
Class Prediction

즉 detection을 하나의 regression 문제로 처리합니다.

2. Grid 기반 예측

이미지를 S × S grid로 나눕니다.

S × S grid

각 grid는 다음 값을 예측합니다.

(x, y, w, h, confidence)

기호 의미

  • x, y : bounding box 중심 좌표
  • w, h : bounding box 크기
  • confidence : 객체 존재 확률

왜 필요한가

모든 detection을 하나의 forward pass로 처리합니다.

3. Bounding Box 표현

YOLO는 bounding box를 다음과 같이 표현합니다.

bbox = (x, y, w, h)

좌표는 grid 기준으로 정규화됩니다.

왜 중요한가

모델이 위치 정보를 학습할 수 있습니다.

4. Confidence Score

confidence는 다음 확률을 의미합니다.

confidence = P(object) × IoU

기호 의미

  • P(object) : 객체 존재 확률
  • IoU : Intersection over Union

왜 필요한가

bounding box의 신뢰도를 평가합니다.

5. IoU (Intersection over Union)

IoU는 두 bounding box의 겹침 정도를 측정합니다.

IoU = Area(overlap) / Area(union)

기호 의미

  • overlap : 두 box 교집합 영역
  • union : 두 box 합집합 영역

왜 중요한가

예측 box의 정확도를 평가합니다.

6. YOLO Loss

YOLO는 다음 요소를 함께 학습합니다.

  • bounding box 위치
  • objectness
  • class prediction

손실 함수 구조:

L = L_bbox + L_obj + L_class

기호 의미

  • L_bbox : 위치 손실
  • L_obj : 객체 존재 손실
  • L_class : 클래스 분류 손실

왜 필요한가

위치와 클래스 정보를 동시에 학습합니다.

7. One-Stage vs Two-Stage Detector

구조 특징
Two-Stage Region proposal 후 classification
One-Stage 한 번에 detection

대표 모델:

방법 예시
Two-Stage Faster R-CNN
One-Stage YOLO, SSD

8. 속도 vs 정확도 Tradeoff

모델 속도 정확도
YOLO 매우 빠름 중간
Faster R-CNN 느림 높음

왜 이런 tradeoff가 발생하는가

  • two-stage는 proposal refinement 가능
  • one-stage는 빠른 inference 중심

9. Non-Maximum Suppression (NMS)

YOLO는 여러 중복 bounding box를 제거해야 합니다.


1 높은 confidence box 선택
2 IoU 높은 box 제거
3 반복

조건:

IoU > threshold → remove

왜 필요한가

중복 detection을 제거합니다.

코드-수식 연결

개념 코드 설명
bbox 예측 model(image) bounding box 예측
IoU 계산 torchvision.ops.box_iou() box overlap 계산
NMS torchvision.ops.nms() 중복 box 제거

자주 하는 오해 5개

  • YOLO는 항상 가장 정확한 detector라고 생각한다
  • one-stage detector는 항상 성능이 낮다고 생각한다
  • grid 하나는 하나의 객체만 예측한다고 생각한다
  • confidence score는 단순 확률이라고 생각한다
  • NMS는 모델 학습 과정이라고 생각한다
